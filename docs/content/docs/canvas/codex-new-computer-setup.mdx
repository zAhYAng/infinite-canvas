# Codex 新电脑连接画布

这份文档可以直接发给新电脑上的 Codex。目标是让 Codex 操作 `https://image.v2api.top/` 无限画布。

## 给 Codex 的任务

你要帮助用户在这台电脑上连接 V2API 无限画布。普通操作画布不需要拉取项目代码，不要要求用户 clone 仓库。只需要安装 Node.js、启动本机 Canvas Agent、给 Codex 配 MCP，然后让网页画布连接本机 Agent。

如果当前 Codex 环境有浏览器控制工具，应优先由 Codex 自己打开网页、填写 Agent 连接信息并点击连接。只有浏览器控制工具不可用、被权限阻止或无法访问当前页面时，才让用户手动连接。

## 前置条件

- 电脑已安装 Node.js。
- 电脑已安装 Codex，并且可以运行终端命令。
- 用户能打开 `https://image.v2api.top/`。

## 一次性配置 MCP

先确认 Codex 已注册 `infinite-canvas` MCP。如果没有，执行：

```powershell
codex mcp add infinite-canvas -- npx -y @basketikun/canvas-agent mcp
```

配置后，新开一个 Codex 对话，让 MCP 工具加载进来。

如果这台电脑已经安装了 `v2api-infinite-canvas` 个人插件，可以跳过手动 MCP 注册。插件里的 MCP 启动脚本会在没有项目源码时自动使用 `npx -y @basketikun/canvas-agent mcp`。

## 启动本机 Canvas Agent

在一个终端里执行：

```powershell
npx -y @basketikun/canvas-agent
```

终端会输出类似：

```text
Infinite Canvas Agent
Local URL: http://127.0.0.1:17371
Connect token: xxxxxxxxx
```

保持这个终端窗口运行，不要关闭。

## 连接网页画布

优先让 Codex 自动完成：

1. 用浏览器控制工具打开 `https://image.v2api.top/`，或用户给出的具体画布 URL。
2. 在网页右上角打开 `Agent -> 本机`。
3. 填入终端输出的 `Local URL`。
4. 填入终端输出的 `Connect token`。
5. 点击连接。

如果 Codex 无法控制浏览器，再让用户按上面同样步骤手动连接。Codex 启动本机 Agent 后，可以从终端输出读取 URL 和 token；必要时也可以读取 `~/.infinite-canvas/canvas-agent.json`。

连接成功后，Codex 应该优先使用 `infinite-canvas` MCP 工具读取和操作画布，例如读取状态、创建节点、连接节点、触发生图流程等。

## 验证连接

Codex 连接后先读取画布状态。如果工具返回“当前没有已连接画布”，让用户检查：

- 本机 Agent 终端是否还在运行。
- 网页 `Agent -> 本机` 是否显示已连接。
- URL 是否是 `http://127.0.0.1:17371`。
- token 是否和终端输出一致。
- 刷新页面后是否需要重新连接。

## 注意事项

- 普通操作画布不需要项目源码。
- 只有开发、部署、修改插件、修改前端代码时才需要拉取 `infinite-canvas` 项目代码。
- 当前画布项目和“我的素材”主要保存在浏览器本地；换电脑后，同一个画布 URL 不一定包含旧电脑里的本地节点数据。需要继续旧画布内容时，先在旧电脑导出或同步画布数据。
- 不要把用户的创作请求发到网页侧边栏里的 Codex 对话；当前 Codex 对话应直接通过 MCP 工具操作画布。
