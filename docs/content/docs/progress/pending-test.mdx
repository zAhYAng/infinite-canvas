## New Manual Checks

- Canvas groups: create a group from the toolbar, drag nodes into and out of it, drag the group with members, copy the group with its children, and delete only the group to confirm the children remain.
- Canvas viewport: create a node after panning away from the origin, use the zoom wheel and slider repeatedly, then use the focus control. Confirm the node stays anchored under the pointer while zooming and the focus control centers the selected node or all nodes when nothing is selected.
- Agent entry: open a canvas project and confirm the global Agent expands on the chat tab. Close it, then use the top-navigation Agent action and confirm it again opens on chat; the tab order must be chat, connection, history, then log.
- Prompt refinement: enter a prompt in both image and video workbenches, choose AI refinement, and confirm the configured text model replaces the draft with a directly usable prompt without creating a visual task. With no valid text-model configuration, confirm it opens configuration instead.
- Site Agent: connect once, switch among canvas, image, video, prompts, and assets, then verify the same conversation and tool confirmation mode remain active. Confirm that `workbench_*_generate` only submits after a `run=true` tool call and the selected confirmation policy permits it.
- Workbench validation: choose a known constrained image/video profile, enter an illegal reference count, duration, or resolution, and confirm no generation request is sent.
- Canvas onboarding: open a canvas in a fresh browser profile and confirm the three animated steps focus the top navigation, open the online Agent composer, and then identify its text-model selector. Confirm the Agent-step controls remain above the composer on desktop and a low mobile viewport, and that Escape or the close control dismisses the guide without blocking normal canvas interactions.
- Home navigation onboarding: open the home page in a fresh browser profile and confirm the prompt showcase is absent. On desktop, confirm the guide highlights each of the five top navigation entries in sequence and its previous, next, skip, and finish controls remain visible; on mobile, confirm it highlights the menu entry and summarizes the available workspaces.
- Agent-led creation: complete the home navigation guide and confirm the final step highlights Agent. With a connected Agent, the panel must open on chat with a focused, editable planning prompt and no request sent automatically; without one, it must open the connection setup. Use the top-bar help control to restart the home and canvas guides without clearing unrelated browser data.
- Video task center: create pending and failed video tasks from both a canvas and the video workbench. Confirm the global or canvas-toolbar task center shows source, model, retry count, elapsed time, and the upstream failure detail, never displays task credentials, and sends "查看任务" or "查看并重试" to the original source page.
- Storyboard confirmation: ask Agent to create a four-shot storyboard and confirm its root node shows four image requests plus one deferred video, image/video model details, duration, resolution, and reference-material guidance before the confirmation click. Confirm no visual task exists before confirmation, and that selecting one completed shot creates only the video task.
- Paid video smoke: with a dedicated low-cost test key only, explicitly set `ALLOW_PAID_VIDEO_SMOKE=yes`, `NEWAPI_SMOKE_BASE_URL`, `NEWAPI_SMOKE_API_KEY`, and `NEWAPI_SMOKE_VIDEO_MODEL`, then run `bun run smoke:video` from `web/`. Confirm `/v1/models` exposes the model, `/v1/videos` returns only a public task ID, polling completes, and `/content` has an MP4 or WebM signature. Do not place the values in shell history, repository files, or logs.
- Grok video recovery: use a controlled temporary upstream `429` or `5xx` response and confirm the request reuses one `Idempotency-Key` for at most three retries. Then return `Job failed, Please try again later.` from a public task and confirm exactly one new task is created after a short delay and its new ID survives refresh. Submit a parameter rejection as well and confirm no retry is made and the UI shows the upstream reason, error code, and a corrective action.

# 待测试

- 画布生图张数与拖拽交互：画布内图片设置、配置节点、在线 Agent 创建配置和实际图片生成请求统一限制在 1-20 张；节点拖拽开始时阻止外层画布指针事件继续处理，需确认拖动节点、新建节点后编辑、连线拖到空白处创建节点和画布平移互不干扰。
- 画布视频多图与提示词编辑：五个 Grok 视频模型统一通过 v2api `/v1/videos` 异步接口提交，参考图按来源使用 `images` URL、data URI 或 multipart 文件，不再改写提示词或拼装聊天兼容字段；需确认 `grok-imagine-video` 与 fast 模型的多图参考生成生效。图片/视频节点底部提示词输入框聚焦时关闭标签高亮叠加层，需确认双击后有光标且可以在输入框中定位、选择并编辑局部文字。
- 画布结构化参考：图片/视频节点提示面板新增参考图条，视频节点新增“文生视频 / 图生视频 / 首尾帧”模式；参考资源保存为带 `kind`、`role`、`asset://` 的结构化引用，旧画布 `references` 会自动迁移，需确认图生视频、首尾帧视频、图片编辑重试和旧项目打开后引用不丢失。
- 画布视频节点：底部生成按钮改为固定显示“生成”并避免与视频设置按钮重叠；Grok 设置按模型限制时长、清晰度和参考图数量，提交时同时发送 `seconds`、`duration`、`size`、`aspect_ratio`、`resolution`，再通过公开任务 ID 轮询并从 `/content` 下载结果。提交草稿与公开任务 ID 会立即保存；刷新或重试只会复用原任务和幂等键，多标签页通过 Web Locks 在锁内读取最新本地项目并复用同一任务，浏览器不支持 Web Locks 时会阻止提交。待完成任务及其来源/参考节点和所属项目不能删除或替换，WebDAV 同步与旧延迟写入也不能覆盖待完成任务；已完成结果会在下一次提交读取项目前先强制落盘，旧标签页不能把它恢复成 pending。停止只结束本地轮询；收到 408/409/425/429、网络无响应或 5xx 时会使用同一幂等键最多自动重试 3 次；4xx 参数、权限和素材错误不重试并给出修正建议；`/content` 必须具有 MP4/WebM 文件签名，空文件或伪装成 `video/mp4` 的 HTML/JSON 会被拒绝。需确认双标签并发、停止、刷新、临时查询错误、429、源素材删除和异常下载内容下的恢复行为。
- Grok 1.5 视频：原生 `grok-imagine-video-1.5` 及 `preview` / `1080p` 兼容别名均走 v2api 异步任务接口并要求一张首帧图；`preview` 保留所选 480p/720p/1080p，`1080p` 别名强制 1080p。需分别确认原生模型和两个别名的图生视频结果。
- 配置与用户偏好：每个模型渠道新增 OpenAI / Gemini 调用格式选择，默认 OpenAI，切换默认格式时同步更新默认 Base URL；渠道 Tab 增加紧凑醒目的提醒，说明模型是否可选需要到“模型”Tab 选择。
- Gemini 调用格式：支持拉取 Gemini 模型列表，并用于文本问答、在线 Agent 工具调用、基础生图和图生图请求。

- v2api 入口：带 handoff 进入无限画布时先显示连接中状态，并等待配置恢复完成；v2api 入口直接跳转画布页，避免配置尚未写入前点击“开始使用”触发登录引导。
